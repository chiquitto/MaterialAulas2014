==

Um validador examina a entrada, e de acordo com seus requisitos, ele produz um resultado booleano.
Se a entrada não cumprir os requisitos, o validador ainda pode fornecer informações sobre quais requisitos são necessários para que a entrada se torne válida.

Ex.: Uma aplicação pode exigir que um LOGIN tenha entre 6 e 12 caracteres de comprimento, e contenha apenas caracteres alfanumericos.
Um validador (ou varios) pode ser utilizado para essa tarefa, e se o LOGIN não cumprir todos os requisitos, seria util o validador informar quais são os requisitos que precisa ser cumprido.

==

Uso básico dos Validator (validadores)

Todos os Validators do Zend implementam a classe Zend_Validate_Interface, que define 2 métodos:

	1. isValid() - O método isValid() executa uma validação sobre um valor fornecido, retornando TRUE se o valor passa por todos os critérios do validador
	
	2. getMessages() - Se o isValid() retornar FALSE, use o método getMessages() para descobrir os motivos da falha da verificação. O método getMessages() retorna uma matriz (ERRO => MENSAGEM_ERRO)
